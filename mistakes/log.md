### [2026-04-17] task-definition.json の修正をコミットせずにデプロイした

**ミスの内容:** `task-definition.json` を修正（ログループ名を `taskflow-backend-logs` に変更）したが、`git commit` をしないまま次のデプロイを実行。GitHub Actions は古いファイルを参照し、ECS タスクが `awslogs-group` 不存在でクラッシュし続けた。

**原因:** 「ローカルで直した = リポジトリに反映された」という誤った思い込み。複数ファイルを並行して編集していたため、どのファイルがコミット済みかの把握が曖昧になった。

**対処:** 正しい `task-definition.json` を commit・push した後、ECS サービスを再デプロイして解消。

**再発防止:** デプロイ（`git push` または GitHub Actions 実行）の前に必ず `git status` を実行し、「意図したファイルが全てステージング/コミット済みか」を確認する。特に CI/CD 経由のデプロイは「push したもの = 実行されるもの」であるため、手元の変更が全て push されているかを確認する習慣をつける。
