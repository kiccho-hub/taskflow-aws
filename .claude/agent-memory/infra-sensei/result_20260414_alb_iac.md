📚 今日の学習ポイント
- **ALB リスナー と ルール の概念**：
  - `aws_lb_listener` = ポート80でHTTPリクエストを受け付けるための「入口」
  - `aws_lb_listener_rule` = 「受け取ったリクエストをどこに転送するか」の条件分岐（パス・ホストヘッダーなど）
  - デフォルトアクションは「どのルールにも引っかからなかった場合の逃げ道」
- **Priority の小さい数値ほど先に評価される**：複数のルールがあるとき、AWS ALBは Priority の昇順（1→99→100）で評価し、最初にマッチしたルールを採用。
- **Fargate 対応の target_type = "ip"**：EC2インスタンスと異なり、Fargateではコンテナが起動するたびにIPが変わるため、IPアドレス登録が必須。

⚠️ 要注意ポイント（苦手分野）
- **ヘルスチェック失敗時の対応フロー**：現在は matcher = "200" で単純に成功判定しているが、本番環境では API レイテンシーや一時的なエラー（500系）への対応を検討する必要がある。
- **ALB ログ出力・WAF統合**：現在はアクセスログが無効化されている（コメントアウト）。本番環境ではセキュリティ・デバッグのため S3 ログの有効化を推奨。
