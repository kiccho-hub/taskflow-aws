## 今日の学習ポイント

- Cognitoの新しいコンソールUI（2024年11月以降）では「アプリケーションを定義」ステップが追加され、ユーザープール作成がより構造的に整理された
- Hosted UI（Managed Login）とカスタム認証UIの違いを理解した
  - Hosted UI: Cognitoが認証ページを提供（設定が楽だが見た目の自由度が低い）
  - カスタムUI: 自分でUIを作りCognito APIを呼び出す（TaskFlowの方式）
- SPAアプリクライアントはクライアントシークレットを持たない設計が正しい（ブラウザはソースが公開されるため、秘密鍵を埋め込めない）

## 要注意ポイント（苦手分野）

- AWSのUI変化が激しいサービス（Cognito・ECS等）では、手順書と実際の画面の差異を常に意識しながら作業する習慣がさらに定着するとなお良い
- ユーザープール作成後の設定確認（特にメール認証・MFA・カスタム属性）を漏れなく行う最終チェック習慣を身につける
