# 全リソース削除手順（IaC フェーズ移行用）

Task 1-12 のコンソール操作で作成したすべてのリソースを削除し、課金ゼロ状態にします。

> **重要：** 削除順序を守ってください。依存関係があるため、順序を間違うとエラーになります。

---

## 削除される対象リソース一覧

- CloudFront ディストリビューション
- CloudWatch（ダッシュボード、アラーム）
- SNS トピック
- S3 バケット（2 つ）
- ECS サービス + クラスター
- ECS タスク定義（backend / frontend）
- ECR リポジトリ（backend / frontend）
- Application Load Balancer + ターゲットグループ
- RDS PostgreSQL データベース + サブネットグループ
- ElastiCache（Valkey）クラスター + サブネットグループ
- Cognito ユーザープール
- NAT Gateway + Elastic IP
- インターネットゲートウェイ（IGW）
- VPC 関連（セキュリティグループ、ルートテーブル、サブネット）
- IAM ロール（ecsTaskExecutionRole）+ OIDC プロバイダー
- CloudWatch Logs（ロググループ）

---

## 削除手順（順序厳守）

### Step 1: CloudFront ディストリビューション を無効化・削除

```
CloudFront コンソール（https://console.aws.amazon.com/cloudfront/v4/home）
→ ディストリビューション一覧から対象（例：d38u58isnyyzc2）を選択
→ ディストリビューションが「有効」状態の場合：
    「無効にする」ボタンをクリック → 確認ダイアログで「無効にする」
    → 「最終更新日時」列に新しいタイムスタンプが表示されるまで待つ（数分）
→ ステータスが「無効」になったら「削除」ボタンをクリック
→ 確認ダイアログで「削除」を選択
```

> **注意：** 削除前に必ず「無効」状態にする必要があります。「無効化」タブでのキャッシュ無効化（Invalidation）は削除には不要です。無効化（Disable）とキャッシュ無効化（Invalidation）は別の操作です。
>
> CloudFront がフラットレート料金プランに加入している場合は、先にプランをキャンセルしてから削除してください。

### Step 2: CloudWatch（ダッシュボード、アラーム）を削除

```
CloudWatch コンソール → 左ナビ「ダッシュボード」
→ 「taskflow-monitoring」を選択
→ 右上「削除」→ 確認して削除
```

```
CloudWatch コンソール → 左ナビ「アラーム」→「すべてのアラーム」
→ 作成したアラーム（CPU、レスポンスタイムなど）をチェックボックスで選択
→ 「アクション」→「削除」→ 確認ダイアログで「削除」
```

### Step 3: SNS トピック を削除

```
SNS コンソール → 左ナビ「トピック」
→ 「taskflow-alerts」を選択（チェックボックスにチェック）
→ 「削除」→ 確認ダイアログに「これを削除」と入力 → 「削除」
```

### Step 4: ECS サービス を削除

```
ECS コンソール → 左ナビ「クラスター」→「taskflow-cluster」を選択
→ 「サービス」タブ → 「taskflow-backend-service」を選択
→ 「削除」ボタンをクリック
→ 確認ダイアログに「delete」と入力 → 「削除」

同様に：
→ 「taskflow-frontend-service」を選択
→ 「削除」→「delete」と入力 → 「削除」
```

> **重要：** タスク数を 0 にするのではなく、**サービス全体を削除**してください。
> サービス削除後、実行中のタスクが停止するまで数分かかる場合があります。

### Step 5: ECS クラスター を削除

```
ECS コンソール → 左ナビ「クラスター」
→ 「taskflow-cluster」を選択
→ 右上「削除クラスター」ボタンをクリック
→ 確認ダイアログに「delete」と入力 → 「削除」
```

### Step 6: ECS タスク定義 を削除

ECS のタスク定義削除は「登録解除（Deregister）→ 削除」の 2 ステップが必要です。

**Step 6-1: 登録解除（INACTIVE 状態にする）**

```
ECS コンソール → 左ナビ「タスク定義」
→ 「taskflow-backend」ファミリーを選択
→ 全リビジョンをチェックボックスで選択
→ 「アクション」→「登録解除」→ 確認して登録解除

同様に：
→ 「taskflow-frontend」ファミリーを選択
→ 全リビジョンをチェックボックスで選択
→ 「アクション」→「登録解除」→ 確認して登録解除
```

**Step 6-2: 削除（INACTIVE になってから実行）**

```
ECS コンソール → 左ナビ「タスク定義」
→ 「taskflow-backend」ファミリーを選択
→ INACTIVE 状態のリビジョンをチェックボックスで選択
→ 「アクション」→「削除」→ 確認して削除

同様に「taskflow-frontend」も削除
```

> **注意：** 登録解除後、削除完了まで最大 1 時間かかる場合があります（完全削除は非同期処理）。
> 削除処理中のタスク定義（DELETE_IN_PROGRESS）を参照中のサービスやタスクは引き続き動作します。

### Step 7: ECR リポジトリ を削除

```
ECR コンソール（Amazon Elastic Container Registry）
→ 左ナビ「リポジトリ」→「プライベート」タブ
→ 「taskflow/backend」を選択（チェックボックス）
→ 「削除」ボタン
→ 確認ダイアログに「delete」と入力 → 「削除」

同様に：
→ 「taskflow/frontend」を選択
→ 「削除」→「delete」と入力 → 「削除」
```

> **注意：** リポジトリを削除すると、中のイメージもすべて削除されます（元に戻せません）。

### Step 8: Application Load Balancer を削除

**Step 8-1: ALB 本体を削除**

```
EC2 コンソール → 左ナビ「ロードバランシング」→「ロードバランサー」
→ 「taskflow-alb」を選択（チェックボックス）
→ 「アクション」→「削除」
→ 確認ダイアログに「confirm」と入力 → 「削除」
```

**Step 8-2: ターゲットグループを削除（ALB 削除後に実施）**

```
EC2 コンソール → 左ナビ「ロードバランシング」→「ターゲットグループ」
→ 「taskflow-backend-tg」を選択（チェックボックス）
→ 「アクション」→「削除」→ 確認して削除

同様に「taskflow-frontend-tg」（または taskflow-tg-frontend など）も削除
```

> **注意：** ALB を削除してもターゲットグループは自動削除されません。ALB 削除後に別途手動で削除してください。
> ターゲットグループはリスナールールに関連付けられていない状態でないと削除できません。ALB を先に削除することでこの依存が解消されます。

### Step 9: RDS データベース を削除

```
RDS コンソール → 左ナビ「データベース」
→ 「taskflow-db」を選択
→ 「アクション」→「削除」
→ 削除確認ダイアログで以下を設定：
    - 「最終スナップショットを作成しますか？」→ チェックを外す（スナップショット不要）
    - 「自動バックアップを保持しますか？」→ チェックを外す
    - 「インスタンスを削除することを了承します」のチェックボックスにチェック
→ テキストボックスに「delete me」と入力
→ 「削除」をクリック
```

> **警告：** 「最終スナップショットを作成しない」を選択すると復旧不可です。本番環境では必ずスナップショットを作成してください。
>
> RDS の削除処理には数分かかります。ステータスが「削除中」から「削除済み」になるまで待ってから次のステップへ進んでください。

### Step 9-2: RDS サブネットグループ を削除

```
RDS コンソール → 左ナビ「サブネットグループ」
→ 「taskflow-rds-subnet-group」（または作成したサブネットグループ）を選択
→ 「削除」→ 確認ダイアログで「削除」
```

> RDS インスタンスの削除が完了してから実施してください（削除中はサブネットグループを削除できません）。

### Step 10: ElastiCache（Valkey）クラスター を削除

```
ElastiCache コンソール → 左ナビ「Valkey キャッシュ」
→ 「taskflow-valkey」の横のラジオボタンを選択
→ 「アクション」→「削除」
→ 確認ダイアログで：
    - 「最終バックアップを作成」のチェックを外す（不要な場合）
    - クラスター名を入力（「taskflow-valkey」）
→ 「削除」をクリック
```

> **注意：** ElastiCache の左ナビメニューは「Redis キャッシュ」ではなく「Valkey キャッシュ」になっています（2024 年以降の新 UI）。
> 一度に削除できるクラスターは 1 つのみです。複数選択すると削除操作が無効になります。
> ElastiCache には停止機能がないため、削除のみ可能です。

### Step 10-2: ElastiCache サブネットグループ を削除

```
ElastiCache コンソール → 左ナビ「サブネットグループ」
→ 「taskflow-elasticache-subnet-group」（または作成したサブネットグループ）を選択
→ 「削除」→ 確認して削除
```

> ElastiCache クラスターの削除完了後に実施してください（削除中はサブネットグループを削除できません）。

### Step 11: Cognito ユーザープール を削除

Cognito はデフォルトで「削除保護」が有効になっています。削除前に無効化が必要です。

**Step 11-1: 削除保護を無効化**

```
Cognito コンソール → 「ユーザープール」
→ 「taskflow-user-pool」を選択
→ 「設定」メニュー（またはタブ）→「削除保護」セクション
→「非アクティブ化」を選択 → 確認ダイアログで確認
```

**Step 11-2: ユーザープールを削除**

```
Cognito コンソール → 「ユーザープール」
→ 「taskflow-user-pool」を選択
→ 右上「削除」ボタン
→ 確認ダイアログに「taskflow-user-pool」と入力 → 「削除」
```

> **注意：** 削除保護が有効なままでは削除できません。また、削除保護を無効化する際に SMS ロールや SES 設定に問題があるとエラーになる場合があります。
>
> 削除後 14 日間は AWS 内部で非アクティブ状態として保持されます（操作はできません）。

### Step 12: NAT Gateway を削除

```
VPC コンソール → 左ナビ「NAT ゲートウェイ」
→ 「taskflow-nat」を選択（チェックボックス）
→「アクション」→「NAT ゲートウェイを削除」
→ 確認ダイアログで「削除」
→ ステータスが「Deleting」→「Deleted」になるまで待つ（2〜5 分）
```

> **重要（コスト）：** NAT Gateway は時間課金（約 $0.045/時 ≈ $33/月）です。削除完了を確認するまで課金が続きます。

### Step 13: Elastic IP を解放

```
VPC コンソール → 左ナビ「Elastic IP アドレス」
→ NAT Gateway に関連付けられていた EIP を選択（「関連付けのない」状態になっているはず）
→「アクション」→「Elastic IP アドレスの解放」
→ 確認ダイアログで「解放」
```

> **注意：** NAT Gateway 削除後、EIP は自動で「未割り当て」状態になります。未割り当ての EIP は課金対象のため、必ず解放してください。

### Step 14: セキュリティグループ を削除

```
VPC コンソール（または EC2 コンソール）→ 左ナビ「セキュリティグループ」
→ 以下を選択して削除（「アクション」→「セキュリティグループを削除」）：
  - taskflow-backend-sg
  - taskflow-alb-sg
  - taskflow-rds-sg
  - taskflow-elasticache-sg
  - taskflow-frontend-sg（存在する場合）
```

> **注意：** デフォルトセキュリティグループ（default）は削除できません（削除しないでください）。
> 他のリソースが参照しているセキュリティグループは削除できません。ECS・RDS・ALB が完全に削除されてから実施してください。

### Step 15: ルートテーブル を削除

```
VPC コンソール → 左ナビ「ルートテーブル」
→ 「taskflow-public-rt」を選択
    → 「サブネットの関連付け」タブ → 「サブネットの関連付けを編集」
    → すべてのサブネットの関連付けを解除 → 「保存」
    → 「アクション」→「ルートテーブルを削除」→ 確認して削除

同様に：
→ 「taskflow-private-rt」を選択
    → サブネットの関連付けを解除 → 「保存」
    → 「アクション」→「ルートテーブルを削除」→ 確認して削除
```

> **注意：** メインルートテーブル（VPC に自動作成されたもの）は削除できません。カスタムルートテーブルのみ削除対象です。

### Step 16: インターネットゲートウェイ（IGW）を削除

```
VPC コンソール → 左ナビ「インターネットゲートウェイ」
→ 「taskflow-igw」を選択
→「アクション」→「VPC からデタッチ」→ デタッチを確認
→ 再度「taskflow-igw」を選択
→「アクション」→「インターネットゲートウェイを削除」→ 確認して削除
```

> **重要：** IGW は VPC にアタッチされたままでは削除できません。必ずデタッチしてから削除してください。

### Step 17: S3 バケット を削除

```
S3 コンソール → バケット一覧
→ 「taskflow-frontend-840854900854-ap-northeast-1」を選択
→「バケットを空にする」ボタンをクリック
→ テキストボックスに「完全に削除」と入力（日本語コンソールの場合）
    ※ 英語コンソールの場合は「permanently delete」
→ 「バケットを空にする」で実行（バージョニング有効の場合はすべてのバージョンも削除される）
→ 空になったら「閉じる」
→ バケット一覧に戻り「taskflow-frontend-xxx」を選択 → 「削除」
→ テキストボックスにバケット名を入力 → 「バケットを削除」

同様に（存在する場合）：
→ 「taskflow-terraform-state」または「taskflow-logs-xxx」なども同様の手順で削除
```

> **注意：** S3 はバケット削除の前に全オブジェクト（すべてのバージョンと削除マーカー）を削除する必要があります。
> バージョニングが有効なバケットの場合、「バケットを空にする」でバージョンと削除マーカーもすべて削除されます。
> 一度削除したバケットは AWS が復旧できません。

### Step 18: CloudWatch Logs を削除

```
CloudWatch コンソール → 左ナビ「ロググループ」
→ 「/ecs/taskflow-backend」を選択（チェックボックス）
→「アクション」→「ロググループを削除」→ 確認

その他のロググループも確認・削除：
  - /ecs/taskflow-frontend（存在する場合）
  - /aws/rds/...（RDS ログ、存在する場合）
  - /aws/elasticache/...（存在する場合）
```

### Step 19: IAM ロール を削除

```
IAM コンソール → 左ナビ「ロール」
→ 検索ボックスに「taskflow」や「ecsTask」と入力してフィルタリング
→ 「ecsTaskExecutionRole」を選択
→ 右上「削除」
→ 確認ダイアログにロール名を入力 → 「削除」

その他のカスタムロール（taskflow-xxx など）も確認・削除：
→ 「taskflow-github-actions-role」なども削除
```

### Step 20: IAM OIDC プロバイダー を削除

```
IAM コンソール → 左ナビ「ID プロバイダー」
→ 「token.actions.githubusercontent.com」を選択
→ 「削除」→ 確認ダイアログで「削除」
```

> GitHub Actions の OIDC 連携用プロバイダーです。Task 11（CI/CD）で作成した場合は削除してください。

### Step 21: VPC サブネット を削除

```
VPC コンソール → 左ナビ「サブネット」
→ 以下を選択（チェックボックス）して削除：
  - taskflow-public-a
  - taskflow-public-c
  - taskflow-private-a
  - taskflow-private-c
→ 「アクション」→「サブネットを削除」→ 確認して削除
```

> **確認：** 削除前に ENI（ネットワークインターフェース）が残っていないか確認してください。
> 左ナビ「ネットワークインターフェース」で「VPC」フィルターを使って確認できます。

### Step 22: VPC を削除

```
VPC コンソール → 左ナビ「お使いの VPC」（Your VPCs）
→ 「taskflow-vpc」を選択
→「アクション」→「VPC を削除」
→ 確認ダイアログで「削除」
```

> **前提確認：** IGW、サブネット、カスタムルートテーブル、セキュリティグループがすべて削除済みであることを確認してください。依存リソースが残っていると削除できません。
> VPC 削除画面で依存リソースの一覧が表示される場合は、そちらを先に削除してから再試行してください。


## 注意事項

### 削除後は復旧不可（スナップショットなし）

- RDS、ElastiCache は最終スナップショットを作成しないため、**復旧は不可**
- ただし、Terraform で再構築可能（IaC フェーズで自動化予定）

### GitHub Secrets は手動削除

`settings/secrets and variables/actions` でも以下を確認・削除：
- `ECR_REGISTRY`
- `AWS_ROLE_ARN`
- `AWS_REGION`
- `S3_BUCKET_NAME`
- `CLOUDFRONT_DISTRIBUTION_ID`

（必要に応じて残してOK、IaC フェーズで Terraform が使用）

### CloudWatch メトリクスの保持期間

CloudWatch メトリクスは 15 ヶ月保持されます。削除後のリソース再作成時に自動で新しいメトリクスが作成されるため問題ありません。

## IaC フェーズ準備

削除完了後、以下の準備をしてください：

1. Terraform ファイル（`.tf`）を `infra/terraform/` ディレクトリに作成
2. `terraform plan` で削除・再作成を確認
3. `terraform apply` でリソースを自動構築

## FAQ

**Q: 削除に失敗しました。何が原因？**

A: 依存関係が残っている可能性があります。以下を確認：
- ECS サービスが完全に削除されたか？（タスク数 0 ではなく完全削除）
- ALB が RDS と VPC に依存していないか？
- セキュリティグループがまだ使用されていないか？
- IGW がまだ VPC にアタッチされていないか？
- ENI（ネットワークインターフェース）が残っていないか？（VPC コンソール → ネットワークインターフェース で確認）

**Q: IAM ロール削除時に「削除できない」エラーが出ました**

A: 他のリソースがロールを参照している可能性があります。以下を確認：
- ECS タスク定義が参照していないか
- CloudWatch Logs が参照していないか

**Q: Cognito ユーザープールが削除できません**

A: 削除保護が有効になっている可能性があります（デフォルトで有効）。
- Cognito コンソール → ユーザープール → 設定 → 削除保護 → 「非アクティブ化」
- SMS ロールや SES 設定に問題があると削除保護の無効化自体がエラーになる場合があります

**Q: ECS タスク定義を削除しようとしたら「INACTIVE にしてください」と言われました**

A: 削除の前に「登録解除（Deregister）」が必要です。Step 6 の手順に従い、まず「アクション → 登録解除」を実行してから削除してください。

**Q: 課金がまだ発生しています**

A: 以下を確認：
- NAT Gateway は削除されたか？（最も高額：約 $33/月）
- ALB は削除されたか？（約 $18/月）
- RDS は削除されたか？
- CloudFront は無効化・削除されたか？
- Elastic IP が未割り当てのまま残っていないか？（$3.6/月）
